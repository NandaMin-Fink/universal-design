Background

Universal Design is an extension of accessible design principles. It incorporates all the aspects of making products, buildings, communications, etc. accessible to people with disabilities we see in accessible design. What makes Universal Design different is that it extends upon this idea by advocating for the widest number of users by preemptively figuring out how all kinds of people will interact with a design. Universal Design expects diverse users to be a consideration throughout the entire design process as opposed to an afterthought. The reality is that this form of design puts a burden on the designer but the return is a singular design that is dynamic and usable by all.

Web Standards

From a technical standpoint web standards are defined as using “structural languages like HTML, XHTML, and XML presentation languages like CSS, object models like the W3C DOM, and scripting languages like JavaScript.” (Beldman).

More simply, this means using specific structure and elements that are recognized by standard-compliant browsers such as I.E.9+, Apple’s Safari 3.0+, Firefox 3+, Chrome, and Opera 9+

Two specific components of web standards are splitting your site design into (x)HTML for structure and content, and CSS for styling and not using tables for layout purposes.

Principles of Universal Design

Equitable Use

  • The design is useful and marketable to people with diverse abilities.
  • Example: Ensuring your website is screen reader compatible, includes alt text for images, etc.

Flexibility in Use

  • The design accommodates a wide range of individual preferences and abilities.
  • Example: Accommodating a user with weak eyesight by allowing them to use their own high-contrast stylesheet to view text better.

Simple and Intuitive Use

  • Use of the design is easy to understand, regardless of the user's experience, knowledge, language skills, or current concentration level.
  • Example: A form that includes clear labels, descends logically, and provides validation feedback for inputs.

Perceptible Information

  • The design communicates necessary information effectively to the user, regardless of ambient conditions or the user's sensory abilities.
  • Example: A video on a website includes captions, a transcript, and sign language translations.

Tolerance for Error

  • The design minimizes hazards and the adverse consequences of accidental or unintended actions.
  • Example: A shopping website that allows one to undo actions and includes confirmation prompts before any purchases.

Low Physical Effort

  • The design can be used efficiently and comfortably and with a minimum of fatigue.
  • Example: Including a skip navigation feature to allow a user with a screen-reader to skip unnecessary information every time they access, refresh, or change pages.

Size and Space for Approach and Use

  • Appropriate size and space is provided for approach, reach, manipulation, and use regardless of user's body size, posture, or mobility.
  • Example: Responsive website design that adjusts for screen size so users can interact comfortably.
